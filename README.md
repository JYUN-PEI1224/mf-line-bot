# マネーフォワード クラウド勤怠 LINE Bot

LINEからマネーフォワード クラウド勤怠の打刻（出勤・退勤・休憩）および勤怠状況の確認ができる連携Botシステムです。

マネーフォワード クラウド勤怠の公式打刻APIは一般公開されていないため、このシステムではWebセッション経由（スクレイピング）で打刻操作を代行します。

## 機能一覧

- **打刻機能**: 出勤、退勤、休憩開始、休憩終了
- **状況確認**: 当日の打刻履歴と現在の状態確認
- **リッチメニュー**: トーク画面下部のメニューボタンからの直感的な操作
- **ユーザー認証**: LINEユーザーごとに個別のマネーフォワード認証情報を保持

## システム要件

- Python 3.11以上
- LINE公式アカウント（Messaging API）
- マネーフォワード クラウド勤怠のアカウント（会社ID、メールアドレス、パスワード）

## セットアップ手順

### 1. LINE公式アカウントの作成と設定

1. [LINE Developersコンソール](https://developers.line.biz/ja/) にログインします。
2. 新規プロバイダーを作成し、その中に「Messaging API」チャネルを作成します。
3. 「チャネル基本設定」タブから **チャネルシークレット** を取得します。
4. 「Messaging API設定」タブから **チャネルアクセストークン（ロングターム）** を発行して取得します。
5. 同タブの「Webhook設定」で、後述のサーバーのURL（例: `https://your-domain.com/webhook`）を設定し、「Webhookの利用」をオンにします。
6. LINE Official Account Managerで、自動応答メッセージをオフにします。

### 2. サーバーのデプロイ

このリポジトリのコードをサーバー（VPS、Render、Heroku、AWSなど）にデプロイします。

#### Dockerを使用する場合（推奨）

```bash
# リポジトリのクローン（またはファイルの配置）
cd mf-line-bot

# .envファイルの作成
cp .env.example .env
nano .env # 取得したLINEの認証情報を設定

# Dockerコンテナの起動
docker-compose up -d
```

#### 手動で起動する場合

```bash
# 依存パッケージのインストール
pip install -r requirements.txt

# .envファイルの作成と設定
cp .env.example .env
nano .env

# サーバーの起動
bash start.sh
```

### 3. リッチメニューのセットアップ

LINEのトーク画面下部に表示されるメニュー（リッチメニュー）を自動生成・設定するスクリプトを実行します。

```bash
# 仮想環境またはコンテナ内で実行
export LINE_CHANNEL_ACCESS_TOKEN="your_token_here"
python setup_richmenu.py
```

※ `setup_richmenu.py` は Pillow を使用してメニュー画像を自動生成し、LINE APIにアップロードします。

## ユーザーの利用開始手順

サーバーが稼働し、LINE Botを友だち追加したら、以下の手順で利用を開始します。

1. LINE Botのトーク画面を開きます。
2. メニューの「設定」ボタンを押すか、「設定」とメッセージを送信します。
3. Botの案内に従って、以下の情報を順に入力します。
   - **会社ID**: マネーフォワード クラウド勤怠のログインURLに含まれるID
   - **メールアドレス**: ログインに使用するメールアドレス
   - **パスワード**: ログインパスワード
4. 登録が完了すると、メニューボタンから打刻ができるようになります。

## セキュリティに関する注意事項

- このシステムは、ユーザーのマネーフォワード勤怠のログインパスワードを預かり、システム内で暗号化（Base64エンコード）して保存します。
- **本番環境で運用する場合は、必ずHTTPS（SSL/TLS）化されたサーバーを使用してください。**
- パスワードの管理をより強固にするため、本番運用時は `user_store.py` を修正し、データベース（PostgreSQL等）や暗号化ライブラリ（Fernet等）を利用することをお勧めします。
