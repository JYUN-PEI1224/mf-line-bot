# Renderの画面メモ

## 画面1: Renderダッシュボード（ログイン後）
- ファイル: qX3jay5CoKU3.png
- 右上に「+ New」ボタンがある
- 左サイドバーにProjects, Blueprints, Environment groupsなど

## 画面2: + New メニュー
- ファイル: pUkXRdQWXMuD.png
- Project / Static Site / Web Service / Private Service / Background Worker / Cron Job / PostgreSQL / Redis / Blueprint
- 「Web Service」を選ぶ

## 画面3: Connect Git provider（GitHub連携前）
- ファイル: nPPHsz68sgvg.png
- タブ: Git Provider / Public Git Repository / Existing Image
- 「Connect Git provider」の画面
- GitHub / GitLab / Bitbucket のボタンがある
- 「GitHub」ボタンを押す

## 画面4: GitHubの認証許可画面
- ファイル: 9zJLpteq57mR.png
- 「Render by Render would like permission to:」
- Verify your GitHub identity / Know which resources you can access / Act on your behalf
- 「Authorize Render」ボタンを押す

## 画面5: GitHubリポジトリのアクセス設定
- ファイル: RUh5qKSGTgUd.png
- 「Update Render's repository access」
- Repository access: Only select repositories
- 「Select repositories」ドロップダウンで対象リポジトリを選ぶ

## 画面6: リポジトリ一覧（Git Provider タブ）
- ファイル: u6YvbxR3ZBkx.png
- 連携後、GitHubのリポジトリ一覧が表示される
- 検索ボックスでリポジトリを検索できる

## 画面7: Connect a repository（リポジトリ選択）
- ファイル: c6XDQTVfaIK2.png
- 「Connect a repository」という見出し
- 検索ボックスにリポジトリ名を入力
- 一覧に「ユーザー名/リポジトリ名」が表示される
- 右側の「Connect」ボタンを押す

## 画面8: Web Service設定画面
- ファイル: qKMCTS17Bauc.png
- Name, Environment(Docker), Region, Branch, Plans を設定
- 「Free」プランを選択（$0/month）
